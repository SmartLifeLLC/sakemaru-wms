# 複数選択UIコンポーネント デザイン仕様

> **対象画面**: `/admin/wms-auto-order-job-controls`「発注・移動候補生成（越前店）」モーダル
> **対象コンポーネント**: `contractor-selection.blade.php` / `warehouse-selection.blade.php`
> **最終更新**: 2026-04-24

---

## 1. 概要

Filament Action モーダル内に埋め込む **Alpine.js ベースの複数選択UIコンポーネント**。
仕入先（~100件）や倉庫（~20件）をチェックボックスで選択し、Livewire プロパティへリアルタイム同期する。

**設計上限**: 1,000件MAX（現状は仕入先~100件、倉庫~20件だが、将来拡張を考慮）

### 3.3 展開パネル

`x-show="expanded" x-collapse` によるアニメーション付き展開。

#### 検索バー

```html
<input type="text"
       x-model="searchQuery"
       placeholder="仕入先コード・名前で検索..."
       class="w-full rounded-lg border-gray-300 text-sm pl-9 shadow-sm
              focus:border-primary-500 focus:ring-1 focus:ring-primary-500
              dark:bg-gray-700 dark:border-gray-600 dark:text-white">
```

- 左に `heroicon-m-magnifying-glass` アイコン（pl-9で余白）
- フィルタリングはクライアントサイド（Alpine.js computed getter）
- コード・名前の両方で部分一致検索（大文字小文字区別なし）

#### 一括操作ボタン（3つ横並び）

| ボタン | 動作 |
|--------|------|
| すべて選択 | `selectAll()` — 全件を selectedIds に設定 |
| すべて解除 | `deselectAll()` — selectedIds を空配列に |
| 表示中を選択/解除 | `toggleAllVisible()` — 検索絞り込み結果のみ対象にトグル |

**スタイル**: 共通で `text-xs px-3 py-1.5 rounded-md bg-gray-100 hover:bg-gray-200`

#### チェックボックスグリッド

- **レイアウト**: `grid grid-cols-2`（2カラム固定）
- **スクロール**: `max-h-96 overflow-y-auto`（384px）※倉庫選択は `max-h-64`（256px）
- **ストライプ**: `index % 4 >= 2` の行に `bg-gray-50 dark:bg-gray-800/50`
- **選択ハイライト**: `bg-primary-50 dark:bg-primary-900/20`
- **罫線**: 各セルに `border-b border-r dark:border-gray-600`

### 3.4 チェックボックスアイテム（1行の構造）

## 4. Alpine.js データモデル

### 4.1 プロパティ

```javascript
x-data="{
    // --- State ---
    searchQuery: '',           // 検索クエリ（リアルタイムフィルタ用）
    allContractors: @js($data), // サーバーから受け取った全データ（immutable）
    selectedIds: @js($ids),     // 選択中のID配列（初期値: 全件選択）
    expanded: false,            // 展開/折りたたみ状態

    // --- Computed ---
    get filteredContractors() { ... },  // searchQueryで絞り込んだリスト
    get selectedCount() { ... },         // selectedIds.length
    get isAllSelected() { ... },         // 全件選択かどうか
    get allVisibleSelected() { ... },    // 表示中の全件が選択済みか

    // --- Methods ---
    isSelected(id) { ... },       // IDが選択中か判定
    toggle(id) { ... },           // 1件トグル
    toggleAllVisible() { ... },   // 表示中を一括トグル
    selectAll() { ... },          // 全件選択
    deselectAll() { ... }         // 全件解除
}"
```

### 4.2 データ同期（x-effect）

```javascript
x-effect="$wire.set('selectedContractorIds', selectedIds, false)"
```

- `false` = Livewire への即時リクエストを抑制（deferred update）
- モーダルの「生成開始」ボタン押下時に Livewire 側で `$this->selectedContractorIds` を参照

### 4.3 データ構造（1アイテム）

```typescript
interface ContractorItem {
    id: number;                    // 仕入先ID (PK)
    code: string;                  // 仕入先コード（表示用）
    name: string;                  // 仕入先名
    transmission_type: 'INTERNAL' | 'UNKNOWN' | string;  // 移動/発注区分
    transmission_type_label: string;  // 区分ラベル
    generation_time: string | null;   // 自動発注時刻 (e.g., "08:00")
}

interface WarehouseItem {
    id: number;                    // 倉庫ID (PK)
    code: string;                  // 倉庫コード
    name: string;                  // 倉庫名
}
```

## 5. Livewire 連携

### 5.1 プロパティ（ListWmsAutoOrderJobControls.php）

```php
// 仕入先選択
public array $selectedContractorIds = [];  // Alpine から同期
public array $contractorsData = [];        // mount時に取得 → Blade へ渡す

// 倉庫選択（管理者メニュー用）
public array $selectedWarehouseIds = [];
public array $warehousesData = [];
```

### 5.2 初期化（mount）

```php
public function mount(): void
{
    // 仕入先: 全件取得 → 全件選択状態で初期化
    $this->contractorsData = $this->getContractorsForWarehouse();
    $this->selectedContractorIds = collect($this->contractorsData)
        ->pluck('id')->values()->toArray();

    // 倉庫: 同様
    $this->warehousesData = $this->getActiveWarehouses();
    $this->selectedWarehouseIds = collect($this->warehousesData)
        ->pluck('id')->values()->toArray();
}
```

### 5.3 Blade 内でのデータ取得

```php
@php
    $lw = $getLivewire();
    $data = $lw->contractorsData ?? [];

    // contractorsData が空の場合、直接取得（フォールバック）
    if (empty($data)) {
        $data = $lw->getContractorsForWarehouse();
        $lw->contractorsData = $data;
        $lw->selectedContractorIds = collect($data)->pluck('id')->values()->toArray();
    }

    $contractorIds = collect($data)->pluck('id')->values()->toArray();
@endphp
```

## 8. 1,000件MAX対応の設計指針

現状の実装には件数上限が存在しない。1,000件MAXに対応する場合の考慮点:

### 8.1 パフォーマンス対策

| 課題 | 現状（~100件） | 1,000件時の対策 |
|------|---------------|----------------|
| 初期描画 | `x-for` で全件レンダリング | **仮想スクロール（virtual scroll）** を導入。表示領域内のDOMのみ生成 |
| 検索フィルタ | `Array.filter()` で即時 | そのまま利用可能（JSの1,000件filterは<1ms） |
| 選択ID同期 | `x-effect` で毎回全配列を `$wire.set` | **debounce（200ms）** を追加。頻繁な`selectAll`操作でのリクエスト抑制 |
| チェックボックス操作 | `splice`/`push` で直接変更 | `Set` ベースに変更してO(1)のhas/add/delete |

### 8.2 UI改善案

| 改善項目 | 内容 |
|---------|------|
| グリッド列数 | 1,000件では **3カラム** を検討（`grid-cols-3`） |
| スクロール高さ | `max-h-96`（384px）→ `max-h-[480px]` に拡大 |
| 選択件数表示 | 上限警告: "1000/1000件 選択中 (最大)" |
| ページネーション | 任意: 100件/ページの仮想ページ切り替え |
| カテゴリ分類 | 仕入先をグループ（例: 地域別）でセクション分け |

### 8.3 コード側の対応ポイント

```javascript
// 現状: Array.includes() は O(n)
isSelected(id) {
    return this.selectedIds.includes(id);  // 1,000件では毎回ループ
}

// 改善案: Set を使用して O(1) に
// Alpine.js は reactive に Set を追跡できないため、
// selectedIds (Array) と selectedIdSet (Set) を二重管理するか、
// Map/Object ベースに切り替える
```

```javascript
// 改善案: debounce付き同期
x-effect="clearTimeout(_t); _t = setTimeout(() => {
    $wire.set('selectedContractorIds', selectedIds, false)
}, 200)"
```

## 10. 仕入先選択と倉庫選択の差分

| 項目 | 仕入先選択 | 倉庫選択 |
|------|-----------|---------|
| Bladeファイル | `contractor-selection.blade.php` | `warehouse-selection.blade.php` |
| Livewire同期先 | `selectedContractorIds` | `selectedWarehouseIds` |
| データソース | `getContractorsForWarehouse()` | `getActiveWarehouses()` |
| スクロール高さ | `max-h-96` (384px) | `max-h-64` (256px) |
| 移動バッジ | あり（INTERNAL時に「移動」表示） | なし |
| 生成時刻表示 | あり（`generation_time`） | なし |
| 検索プレースホルダ | 仕入先コード・名前で検索... | 倉庫コード・名前で検索... |
| 0件メッセージ | 仕入先が登録されていません | 倉庫が登録されていません |
| 利用モーダル | 全3モーダルで共用 | 管理者メニュー「仕入先別発注候補生成」のみ |
