# 入庫処理 Android アプリ - UI/UX 仕様書

## 概要

本ドキュメントは、入庫処理システムの Android Native アプリ実装に必要な画面構成、画面遷移、ボタン配置について定義します。

## 対象端末

- **BHT-M60** ハンディターミナル
- 解像度: 480×800 (WVGA)
- 物理ファンクションキー: F1〜F4
- バーコードリーダー内蔵

---

## 画面一覧

| # | 画面名 | 画面ID | 説明 |
|---|--------|--------|------|
| 1 | ログイン画面 | `login` | ピッカーコード・パスワード入力 |
| 2 | 倉庫選択画面 | `warehouse` | 作業倉庫選択 |
| 3 | 商品リスト画面 | `list` | 入庫予定商品一覧（検索機能付き） |
| 4 | スケジュール一覧画面 | `process` | 選択商品の入庫予定一覧 |
| 5 | 入力画面 | `input` | 数量・賞味期限・ロケーション入力 |
| 6 | 履歴画面 | `history` | 当日の入庫作業履歴 |

### 5. 入力画面 (`input`)

#### レイアウト

```
┌────────────────────────────┐
│ ← 華むすびの蔵センター 入庫処理 │
├────────────────────────────┤
│ 4901234567890 (10001)      │  ← 商品情報
│ 純米大吟醸 雪月花 720ml    │
├────────────────────────────┤
│ 入荷日: 2026-01-20         │  ← amber背景
├────────────────────────────┤
│ 📅 賞味期限 (任意)         │
│ [    2027-01-20        ]   │
├────────────────────────────┤
│ 📍 ロケ                    │
│ [🔍 A-01-001           ]   │
│ ┌──────────────────────┐   │  ← ドロップダウン
│ │ A 01 001             │   │
│ │ A 01 002             │   │
│ └──────────────────────┘   │
├────────────────────────────┤
│ 📦 入庫予定 : 50           │
│ [         40           ]   │  ← 数量入力
├────────────────────────────┤
│  F1:自動 | F2:登録 | F3:-- | F4:戻る │
└────────────────────────────┘
```

#### 入力フィールド

| フィールド | タイプ | 必須 | 初期値 |
|-----------|-------|-----|-------|
| 賞味期限 | date | ○ | スケジュールの expiration_date または商品のデフォルト |
| ロケーション | text + autocomplete | × | default_location.display_name |
| 入庫数量 | number | ○ | expected_quantity |

#### フィールドナビゲーション

- Tab/↓: 次のフィールドへ
- Shift+Tab/↑: 前のフィールドへ
- フィールド順序: 賞味期限 → ロケーション → 数量

#### ロケーション検索

- 1文字以上入力で `/api/incoming/locations` を検索
- 300ms debounce
- ドロップダウンで候補表示
- 選択で `location_id` と `location_search` をセット

#### バリデーション

| 項目 | ルール |
|-----|-------|
| 数量 | 0 < qty <= expected_quantity |

#### ファンクションキー

| キー | ラベル | アクション | 条件 |
|-----|-------|----------|-----|
| F1 | 自動 | 数量を予定数にセット | - |
| F2 | 登録 | 入庫確定処理 | canSubmit = true |
| F4 | 戻る | スケジュール一覧 or 履歴へ戻る | - |

#### 登録処理フロー

1. 作業データがなければ `POST /api/incoming/work-items` で作成
2. `PUT /api/incoming/work-items/{id}` で更新
3. 新規の場合のみ `POST /api/incoming/work-items/{id}/complete` で完了
4. 商品リストを再取得
5. 同じ商品のスケジュール一覧へ遷移

## 共通コンポーネント

### ヘッダー

```
┌────────────────────────────┐
│ ← {画面タイトル}            │
└────────────────────────────┘
```

- 背景: `bg-gray-800`
- テキスト: white, bold
- 高さ: 40px
- 左に戻るボタン（←）

### フッター（ファンクションキーバー）

```
┌────────────────────────────┐
│  F1:xxx | F2:xxx | F3:xxx | F4:xxx │
└────────────────────────────┘
```

- 背景: `bg-gray-800`
- 高さ: 40px
- 4分割均等配置
- 各ボタンは上段にキー名（yellow-400）、下段にラベル

### ローディングオーバーレイ

```
┌────────────────────────────┐
│                            │
│         ⏳ 処理中...        │
│                            │
└────────────────────────────┘
```

- 半透明黒背景
- 中央にスピナー + メッセージ

### 通知トースト

```
┌────────────────────────────┐
│ ✓ 入庫が完了しました        │
└────────────────────────────┘
```

- 画面下部に表示
- 3秒後に自動非表示
- タイプ別カラー: success(green), error(red), info(blue)

## キーボード・バーコード対応

### 物理キーマッピング

| 物理キー | アクション |
|---------|----------|
| F1 | 画面ごとの F1 アクション |
| F2 | 画面ごとの F2 アクション |
| F3 | 画面ごとの F3 アクション |
| F4 | 画面ごとの F4 アクション |
| ↑ | リストで前の項目 / 入力フォームで前のフィールド |
| ↓ | リストで次の項目 / 入力フォームで次のフィールド |
| Enter | 選択確定 |
| Tab | 次のフィールドへ |
| Shift+Tab | 前のフィールドへ |

### バーコードスキャン

| 画面 | スキャン対象 | 処理 |
|-----|------------|------|
| 商品リスト | JANコード | 検索バーに値をセット → 自動検索 |
| 入力画面 | ロケーションバーコード | ロケーション検索 → 自動選択 |

## API 呼び出しタイミング

| 画面遷移 | API 呼び出し |
|---------|------------|
| ログイン成功後 | `GET /api/master/warehouses` |
| 倉庫選択後 | `GET /api/incoming/schedules` |
| 商品リストで検索 | `GET /api/incoming/schedules?search=xxx` |
| 商品リストに戻る | `GET /api/incoming/work-items?status=WORKING` |
| 履歴画面表示 | `GET /api/incoming/work-items?status=all&from_date=today` |
| スケジュール選択 | (API呼び出しなし) |
| 入庫登録 | `POST /work-items` → `PUT /work-items/{id}` → `POST /work-items/{id}/complete` |
| ロケーション入力 | `GET /api/incoming/locations?search=xxx` |
