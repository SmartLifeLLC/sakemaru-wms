現在の自動発注の流れ、
1. 発注・移動候補生成
admin/wms-auto-order-job-controls
2. 移動候補処理
admin/wms-stock-transfer-candidates
3. 発注候補処理
admin/wms-order-candidates
4. 発注・移動確定待ち
admin/wms-order-confirmation-waiting


課題1. 自動発注計算は仕入れ先別にタイミングが異なる。仕入先別に時間になったら自動発注ができるようにする。
課題2. 手動強制発注・手動強制移動が必要。現在は上記のながれに沿って発注計算と処理が必要で、その途中に発注、移動の生成ができない。
強制的に発注・移動をし発注データ、移動データを生成する場合がある。自動発注と手動の発注生成が必要。いまも、発注候補・移動候補の生成ができるが、フローに沿って、対応する必要がある。
発注作業中に別の人が今すぐ必要な発注データの生成ができない。これを修正したい。


改善案
- 1. wms_contractor_settingsに自動発注生成時刻を設定. この時刻になったら、まだ発注生成を実施してない発注先に対して自動発注データを生成する。
- 2. 自動発注データに(移動候補・発注候補)に対して、生成したデータの状態をどこまで確定するかの制御が必要。wms_contractor_warehouse_settingsに持たせる。
 == 移動候補については全て承認後確定待ちにする
 == 発注データに対しては三つの対応が必要で、これらがwms_contractor_warehouse_settingsの管理対象になる。 

status1 : 既存のまま、移動候補・発注候補に表示するもの（移動候補一覧・発注候補一覧に出る状態）
status2 : 発注承認までおこなうもの。（発注、移動確定待ち状態）
status3 : 発注確定までおこなうもの。（入荷候補まで作成される）


- 3. 発注確定データに対して、キャンセルもできるようにしたい。この場合入荷予定データの削除が必要
