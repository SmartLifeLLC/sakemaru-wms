## 6. 実装変更内容

### 6.1 データベース変更
- `wms_order_candidates`テーブルに`ordering_code`カラムを追加（VARCHAR(13)）

### 6.2 ロジック変更

#### OrderCandidateCalculationService
- 発注候補生成時に`item_search_information`の`is_used_for_ordering=true`のコードを取得
- 取得したコードを13桁にゼロパディングして`ordering_code`に保存

#### HanaOrderFileGenerator
- Dレコード生成時、候補の`ordering_code`を優先使用
- `ordering_code`がnullの場合はフォールバック処理を実行

### 6.3 フォールバックロジック
1. `is_used_for_ordering=true`のコードを検索
2. なければ`code_type='JAN'`（PIECE優先）を検索
3. なければ`code_type='OTHER'`（数字のみ7桁以上）を検索
4. 取得したコードを13桁にゼロパディング
