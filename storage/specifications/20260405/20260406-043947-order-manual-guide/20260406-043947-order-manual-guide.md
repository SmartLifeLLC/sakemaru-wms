# 発注業務マニュアル（解説ページ拡充）

- **作成日**: 2026-04-06
- **ステータス**: ドラフト

## 背景・目的

現在の解説ページ（`admin/auto-order-guide`）はシステム操作の概要・ステータス・画面リンクが中心で、**日常業務としてどの順番で何をすべきか**のオペレーションマニュアルが不足している。

新規担当者が朝の発注業務を迷わず実施できるよう、時系列に沿った操作手順書を追加する。

## 現状の実装

### 解説ページ構造（auto-order-guide.blade.php）

| タブ | 内容 |
|------|------|
| 概要 | 全体の流れ（6ステップ）、移動と発注の関係、ステータス一覧、関連画面リンク |
| メニュー詳細 | 生成・移動・発注・確定・履歴のサブタブ |
| 設定 | マスタ設定画面へのリンク集 |

### メガメニュー

`解説` タブ → `発注` カテゴリ → `自動発注`（admin/auto-order-guide）

### 発注業務の実際の流れ

```
09:30  HUB倉庫の締切の早い仕入先の発注を実施
       ↓
11:30  各サテライト倉庫の発注を実施
       ↓
12:00  HUB倉庫の発注を実施
```

### 倉庫構成

| 区分 | 倉庫 | 特徴 |
|------|------|------|
| HUB | 華むすびの蔵センター（91） | メインセンター、全店舗への移動出荷元 |
| HUB | オレンジ冷凍倉庫（101） | 外部倉庫だがHUBと同じ動き。冷凍品専門 |
| サテライト | 本店(1), 二の宮店(2), 坂井店(3) 等15店舗 | 各店舗、HUBから在庫移動を受ける |

### 注意点

1. **オレンジ倉庫の特殊性**
   - 外部倉庫だが内部HUB倉庫と同じ運用
   - サテライト倉庫はオレンジ倉庫にも在庫移動依頼を出す
   - オレンジ倉庫は波動出荷（ピッキング→出荷指示）が必要
   - ピッキングリストを出力し、オレンジ倉庫に出荷指示
   - ウェブ上でピッキング実施（実際の在庫の移動処理）
   - オレンジ倉庫の在庫管理が必要

2. **サテライト→HUBの発注順序**
   - サテライト倉庫の移動発注が未実施でもHUB倉庫の発注は可能
   - ただしその場合、移動出荷分は考慮されない（在庫計算に含まれない）

## 変更内容

### 概要

解説ページに「発注の流れ」タブを追加し、時系列に沿った操作マニュアルを提供する。各ステップに操作画面・ボタン名・注意点を明記する。

### 詳細設計

#### 新タブ「発注の流れ」の構成

既存の3タブ（概要・メニュー詳細・設定）に加え、先頭に「発注の流れ」タブを挿入する。

```
[発注の流れ] [概要] [メニュー詳細] [設定]
```

#### タブ内セクション

##### セクション1: 1日の発注スケジュール

タイムラインUI（縦のステップカード）で以下を表示:

| 時刻 | 操作 | 画面 | 対象倉庫 |
|------|------|------|----------|
| 09:30 | 締切の早い仕入先の発注候補生成 | 発注・移動候補生成 | HUB（華むすびの蔵） |
| 09:30〜11:30 | 各サテライト倉庫の発注候補生成→承認→確定 | 発注候補生成→候補一覧→確定待ち | 各サテライト店舗 |
| 11:30〜12:00 | HUB倉庫の発注候補生成→承認→確定 | 発注候補生成→候補一覧→確定待ち | HUB（華むすびの蔵 + オレンジ） |
| 12:00〜 | 波動生成→ピッキングリスト出力 | 波動管理 | オレンジ倉庫 |

##### セクション2: サテライト倉庫の発注手順（ステップバイステップ）

各ステップをカード形式で:

**Step 1: 発注・移動候補生成**
- 画面: `admin/wms-auto-order-job-controls`
- 操作:
  1. トップバーで対象の倉庫を選択（例: 二の宮店）
  2. 「発注・移動候補生成」ボタンをクリック
  3. 仕入先を確認（デフォルト全選択）、必要に応じて絞り込み
  4. 「生成開始」をクリック
- 注意: 発注ステータスWidget で各倉庫の完了状況を確認可能

**Step 2: 移動候補の確認・承認**
- 画面: `admin/wms-stock-transfer-candidates`
- 操作:
  1. 倉庫タブで対象倉庫を選択
  2. 移動候補の数量・移動先を確認
  3. 問題なければ一括承認、不要なものは除外
- 注意: 移動数量を変更すると発注候補の数量が自動再計算される

**Step 3: 発注候補の確認・承認**
- 画面: `admin/wms-order-candidates`
- 操作:
  1. 倉庫タブで対象倉庫を選択
  2. 発注数量・入荷予定日を確認（テーブル上で直接編集可能）
  3. 一括承認

**Step 4: 発注・移動確定**
- 画面: `admin/wms-order-confirmation-waiting`
- 操作:
  1. 移動確定待ちタブで承認済み移動候補を確認
  2. 発注確定待ちタブで承認済み発注候補を確認
  3. 「発注・移動確定」ボタンをクリック
  4. 処理完了まで待機（プログレスバー表示）
- 結果: 移動伝票生成 + 発注CSVファイル生成 + 入荷予定作成

##### セクション3: HUB倉庫の発注手順

サテライトと同じStep 1〜4だが、以下の違いを強調:

- HUB倉庫発注時、サテライト倉庫の発注が未完了の場合はモーダルに警告表示
- サテライトの移動出荷分を考慮した在庫計算が行われるため、サテライト先行が推奨

##### セクション4: オレンジ倉庫の出荷手順

**Step 5: 波動生成**
- 画面: `admin/waves`
- 操作:
  1. 倉庫「オレンジ冷凍倉庫」を選択
  2. 出荷日を選択
  3. 配送コースを選択（移動伝票のコースが表示される）
  4. 「波動生成」をクリック

**Step 6: ピッキング**
- 画面: `admin/wms-picking-tasks`
- 操作:
  1. ピッキングリストを出力
  2. オレンジ倉庫に出荷指示
  3. ウェブ上でピッキング完了処理

##### セクション5: よくある質問（FAQ）

カード形式で:
- Q: サテライト倉庫の発注をせずにHUB倉庫の発注をしてもよいか？
  - A: 可能。ただし移動出荷分が在庫計算に含まれない
- Q: 発注候補の数量を変更したい場合は？
  - A: 承認前であればテーブル上で直接編集可能
- Q: 誤って確定してしまった場合は？
  - A: 確定済み画面から確認。JX送信前であれば対応可能
