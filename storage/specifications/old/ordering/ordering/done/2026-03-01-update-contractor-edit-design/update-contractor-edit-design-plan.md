# 発注先編集ページ デザイン変更 作業計画

## 前提

- 発注先編集ページ（`admin/contractors/{id}/edit`）は Filament 4 の EditRecord + combinedRelationManagerTabsWithContent で構成
- 現在のタブ構成: **編集** / 仕入先 / 倉庫別納品可能日 / 倉庫別納入先指定コード
- 「編集」タブ内に4セクション: 基本情報 / 設定 / 発注メール設定(collapsible) / WMS送信設定(collapsible)
- 先行作業で「送信先発注先」→「発注データ集約先」へラベル名変更済み

## P1: タブ名変更＆セクション統合

### 目的

- 「編集」タブを「基本情報」に変更
- 「設定」セクション（lead_time_id, is_active）のフィールドを「基本情報」セクションに統合し、「設定」セクションヘッダーを削除

### 修正対象ファイル

| ファイル | 変更内容 |
|---------|---------|
| `app/Filament/Resources/Contractors/Pages/EditContractor.php` | `contentTabLabel` を「基本情報」に変更 |
| `app/Filament/Resources/Contractors/Schemas/ContractorForm.php` | 「設定」セクションのフィールドを「基本情報」セクションに移動 |

### 修正方針

**EditContractor.php:**
- `protected ?string $contentTabLabel = '編集';` → `'基本情報'` に変更

**ContractorForm.php:**
- 現在の「基本情報」セクション（3カラムGrid内に9フィールド）に「設定」セクションの `lead_time_id` と `is_active` を追加
- 「設定」セクション（Section::make('設定')）を削除
- Grid(3) の中に lead_time_id と is_active を追加する（基本情報の末尾に配置）

### 完了条件

- [ ] タブ名が「基本情報」と表示される
- [ ] 「設定」セクションヘッダーが表示されない
- [ ] lead_time_id と is_active が基本情報セクション内に表示される
- [ ] 構文エラーなし

## P2: WMS送信設定の右側配置＆ヘルパーテキスト修正

### 目的

- 基本情報セクションの右側にWMS送信設定を表示（2カラムレイアウト）
- 発注データ集約先のヘルパーテキストを更新

### 修正対象ファイル

| ファイル | 変更内容 |
|---------|---------|
| `app/Filament/Resources/Contractors/Schemas/ContractorForm.php` | レイアウト変更、ヘルパーテキスト修正 |

### 修正方針

**レイアウト:**
- Grid(2) で左右に分割
  - 左カラム: 基本情報セクション（既存の基本情報フィールド + 統合した設定フィールド）
  - 右カラム: WMS送信設定セクション（collapsible解除）
- WMS送信設定はセクション名を維持し、collapsible を解除

**ヘルパーテキスト修正:**
- `wms_transmission_contractor_id` の helperText を変更:
  - 現在: `別の発注先の送信設定を使用する場合に指定`
  - 変更後: `指定した発注先の送信データに本発注先の発注データを集約する（一つのファイルで送信したい場合）`

### 完了条件

- [ ] 基本情報（左）とWMS送信設定（右）が横並びで表示される
- [ ] 発注データ集約先のヘルパーテキストが更新されている
- [ ] WMS送信設定の条件付き表示（JX_FINET時のJX設定表示など）が正常動作
- [ ] 構文エラーなし

## P3: 発注メール設定を別タブに分離

### 目的

- 発注メール設定セクションを「基本情報」タブから分離し、独立したタブとして表示

### 修正対象ファイル

| ファイル | 変更内容 |
|---------|---------|
| `app/Filament/Resources/Contractors/Schemas/ContractorForm.php` | 発注メール設定をタブコンポーネントとして分離 |
| `app/Filament/Resources/Contractors/Pages/EditContractor.php` | タブ構成の調整（必要に応じて） |

### 修正方針

**アプローチ:** Filament 4 の Tabs コンポーネントを使用してフォーム内にタブを作成

- ContractorForm の schema を Tabs コンポーネントで構成:
  - Tab 1「基本情報」: 基本情報 + WMS送信設定（P1, P2で構成済み）
  - Tab 2「発注メール設定」: 現在の発注メール設定セクションの内容

- EditContractor.php の `contentTabLabel` は引き続き使用（RelationManagerタブとの区別用）
  - ただし contentTabLabel は外側のタブ名なので、内側のフォームタブと混同しないよう名称を再検討

**注意:** combinedRelationManagerTabsWithContent() の「コンテンツ」タブの中にさらにTabsを入れ子にする形になる。外側タブ名は内容に合わせて調整する。

### 完了条件

- [ ] 「基本情報」タブと「発注メール設定」タブが分かれて表示される
- [ ] 発注メール設定のフィールド（メールアドレス、タイトル、本文、変数ヘルプ）が正常表示
- [ ] 仕入先/倉庫別納品可能日/倉庫別納入先指定コードの既存タブも正常動作
- [ ] 保存時に全フィールドのデータが正しく永続化される
- [ ] 構文エラーなし

## 制約（厳守）

1. **データベース破壊コマンド禁止**: migrate:fresh, migrate:refresh, db:wipe 等は絶対に実行しない
2. **FK禁止**: データベースの外部キー制約は使用しない
3. **Filament 4 API**: `Filament\Schemas\Components\Section`, `Filament\Schemas\Components\Grid` 等の正しいインポートパスを使用
4. **保存ロジック変更禁止**: EditContractor.php の `mutateFormDataBeforeFill` / `afterSave` のデータマッピングロジックは変更しない（フィールド名は維持）
5. **テーブルデザイン仕様準拠**: コード系ラベルは「CD」表記、アクションボタンは右固定
